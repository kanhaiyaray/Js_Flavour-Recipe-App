## вңЁ Overview

**Flavour** is a full-stack-style **Single Page Application** built entirely with vanilla **HTML5, CSS3, and JavaScript ES6+** вҖ” no frameworks, no backend, no build tools required. It combines a **live recipe discovery engine** powered by the TheMealDB public REST API with a complete **client-side user authentication system** using the browser's `localStorage`.

> рҹ’Ў This project proves that a **production-quality user experience** вҖ” with auth flows, live API data, animated UI, and responsive design вҖ” can be built using only core web technologies. No React. No Node.js. No npm.

### рҹҺҜ Why This Project Stands Out

| Aspect | What Makes It Special |
|--------|----------------------|
| рҹ”җ Auth System | Full Signup вҶ’ Login вҶ’ Logout вҶ’ Session persistence without any backend |
| рҹҺЁ UI Design | Premium dark-gold aesthetic with CSS-only animations and transitions |
| вҡЎ Performance | Under 5KB JS В· Zero HTTP requests on load except fonts and API |
| рҹҢҗ Real Data | Live recipe data from a real public REST API (not hardcoded/mocked) |
| рҹ“Ұ Zero Config | Clone вҶ’ Open вҶ’ Done. No terminal commands needed |

## рҹҺҜ Features

### рҹ”җ Authentication
- вң… **User Registration** вҖ” Name, email, password with validation
- вң… **User Login** вҖ” Email + password credential matching
- вң… **User Logout** вҖ” Clears session with farewell toast
- вң… **Session Persistence** вҖ” Stays logged in after browser close/reopen
- вң… **Duplicate Email Check** вҖ” Inline error on existing accounts
- вң… **Password Validation** вҖ” Minimum 6 character enforcement
- вң… **Inline Error Messages** вҖ” No alert popups, errors shown in form

### рҹҚҪпёҸ Recipe Discovery
- вң… **6 Cuisine Filters** вҖ” Indian, Canadian, American, Thai, British, Russian
- вң… **Live Search** вҖ” Search any dish name, fires on Enter key
- вң… **Recipe Count** вҖ” Shows total results per cuisine/search
- вң… **No Results State** вҖ” Friendly empty state for failed searches
- вң… **Lazy Loading Images** вҖ” Images load only when in viewport

### рҹҺЁ UI & UX
- вң… **Premium Dark Theme** вҖ” Deep black with warm gold accents
- вң… **Animated Auth Modal** вҖ” Slide-up with backdrop blur
- вң… **Skeleton Loaders** вҖ” Shimmer placeholders during API fetch
- вң… **Staggered Card Animations** вҖ” Cards enter with cascading delay
- вң… **Toast Notifications** вҖ” Non-intrusive feedback for all actions
- вң… **Hover Effects** вҖ” Cards lift with shadow on hover
- вң… **Fully Responsive** вҖ” Mobile, tablet, and desktop layouts

## рҹ”җ Authentication System

> **Built entirely on the browser's `localStorage` вҖ” no backend, no server, no database.**

### рҹ“қ Sign Up Flow

```
User fills form (name + email + password)
         в”Ӯ
         в–ј
Validate: password.length >= 6 ?
         в”Ӯ
         в”ңв”Җв”Җ вқҢ No  вҶ’ showError("Password must be at least 6 characters")
         в”Ӯ
         в–ј
Read users array from localStorage
         в”Ӯ
         в–ј

         в”Ӯ
         в”ңв”Җв”Җ вқҢ Yes вҶ’ showError("This email is already registered")
         в”Ӯ
         в–ј

         в”Ӯ
         в–ј

Save: localStorage.setItem("currentUser", newUser)
         в”Ӯ
         в–ј
closeAuth() вҶ’ checkAuthState() вҶ’ showToast("Welcome! рҹҺү")
```

### рҹ”‘ Login Flow

```

         в”Ӯ
         в–ј

         в”Ӯ

         в”Ӯ
         в–ј

```

      в”Ӯ

      в”Ӯ

      в–ј

### рҹ”„ Session Persistence

```js
// Runs on EVERY page load вҖ” restores session automatically
function checkAuthState() {
  const user = JSON.parse(localStorage.getItem("currentUser"));
  if (user) {
    // Show: avatar + greeting + logout button
    // Hide: login/signup buttons
  } else {
    // Show: login/signup buttons
    // Hide: user info
  }
}
```

> рҹ”’ **Security Note:** Passwords are stored in plain text in `localStorage`.
> This is intentional for a **frontend learning project** but is **never acceptable

## рҹҢҗ API Integration

### Provider: [TheMealDB](https://www.themealdb.com/api.php)
Free, no-auth-required public REST API for meal/recipe data.

### Endpoints Used

| Method | Endpoint | Purpose | Example |
|--------|----------|---------|---------|
| `GET` | `/filter.php?a={area}` | Fetch meals by cuisine | `?a=indian` |
| `GET` | `/search.php?s={name}` | Search meals by name | `?s=chicken` |

### Fetch Pattern (async/await)

```js
const fetchData = async (area) => {
  showSkeletons();                          // Show loading UI
  const res  = await fetch(`...${area}`);  // HTTP GET request
  const data = await res.json();           // Parse JSON body
  renderCards(data.meals, area);           // Update DOM
};
```

### Recipe Card Rendering
```js

    <div class="card-body">
      <div class="card-name">${meal.strMeal}</div>
      <span class="card-tag">${area}</span>
    </div>
  </div>
`).join("");
```

### Skeleton Loading
```js
function showSkeletons(count = 6) {
  foodGrid.innerHTML = Array(count)
    .fill('<div class="skeleton"></div>')
    .join("");
}
// Called before every API fetch to show loading state immediately
```
